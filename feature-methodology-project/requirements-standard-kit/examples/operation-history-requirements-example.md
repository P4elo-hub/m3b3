#### Анализ требований — Получить историю операций

**Источник требования:**

| Параметр | Значение |
|----------|----------|
| Фича | Получить историю операций |
| Источник | Product request |
| Инициатор | Product Owner мобильного банка |
| Дата / версия | Draft 2026-05 |

**Стейкхолдеры:**

| Стейкхолдер | Роль | Интерес / ожидание | Влияние / ответственность |
|-------------|------|--------------------|---------------------------|
| Клиент мобильного банка | Основной пользователь | Видит актуальную историю операций на экране | Использует результат фичи |
| Product Owner мобильного банка | Владелец продукта | Получить единый экран истории операций из разных источников | Принимает функциональный результат |
| Команда BDUI | Владелец экранного сервиса | Получить агрегированные данные для формирования экрана | Формирует экран и отдает его фронту |
| Команда backend истории операций | Владелец агрегации | Собрать операции из источников и вернуть согласованный набор данных | Реализует агрегацию и обработку ошибок |
| Владельцы источников операций | Поставщики данных | Передать операции в согласованном формате | Отвечают за полноту и корректность данных источников |

**Бизнес-правила и ограничения:**

| ID | Тип | Описание | Источник / обоснование |
|----|-----|----------|------------------------|
| BR-01 | Бизнес-правило | История операций должна формироваться из шести источников данных по операциям | Product request |
| BR-02 | Бизнес-правило | Backend агрегирует полученные сообщения перед передачей данных в BDUI | Product request |
| BR-03 | Бизнес-правило | BDUI формирует экран истории операций на основе агрегированных данных | Product request |
| CN-01 | Ограничение | Обмен между backend и источниками выполняется асинхронно через Kafka | Product request |
| CN-02 | Ограничение | Поведение при недоступности одного или нескольких источников не определено | GAP-REQ-001 |

**Цель фичи:**

| Вопрос | Ответ |
|--------|-------|
| Какую бизнес-проблему решает? | Сейчас пользователю нужен единый экран истории операций, но данные операций находятся в нескольких источниках |
| Какую ценность приносит пользователю? | Пользователь получает собранную историю операций в одном экране без ручного перехода между источниками |
| Какие метрики улучшает? | Полнота отображения истории операций, успешность загрузки экрана, время получения истории операций. Целевые значения: GAP-REQ-002 |

**SMART-проверка цели:**

| Критерий | Описание | Значение |
|----------|----------|----------|
| Specific | Конкретная | Реализовать получение истории операций через BDUI и backend с агрегацией данных из шести источников |
| Measurable | Измеримая | Измеряется успешностью загрузки экрана, полнотой данных и временем ответа. Целевые значения: GAP-REQ-002 |
| Achievable | Достижимая | Достижимость зависит от готовности Kafka-взаимодействий и контрактов шести источников |
| Relevant | Релевантная | Фича напрямую влияет на пользовательский доступ к истории операций |
| Time-bound | Ограниченная по времени | GAP-REQ-003 |

**Противоречия и решения:**

| Требование 1 | Требование 2 | Источник 1 | Источник 2 | Решение | Обоснование |
|--------------|--------------|------------|------------|---------|-------------|
| Противоречия не выявлены | - | Product request | - | - | На текущем уровне постановки конфликтующих требований нет |

**Gaps и допущения:**

| ID | Тип | Где обнаружено | Что не хватает / что предполагается | Как закрыть |
|----|-----|----------------|-------------------------------------|-------------|
| GAP-REQ-001 | Gap | Бизнес-правила и ограничения / CN-02 | Не определено поведение при недоступности одного или нескольких источников операций | Уточнить у Product Owner и backend команды: частичный результат, ошибка или ожидание всех источников |
| GAP-REQ-002 | Gap | Цель фичи / SMART Measurable | Не указаны целевые значения метрик полноты, успешности загрузки и времени ответа | Добавить целевые значения или способ измерения |
| GAP-REQ-003 | Gap | SMART Time-bound | Не указан срок, релиз или контрольная дата | Добавить релиз, квартал или дату проверки результата |
