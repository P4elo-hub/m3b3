#### Activity Diagram — Вывод средств

Канонические примеры Activity Diagram из `standarts_features (3).md`: базовый процесс, процесс с условиями и варианты параллельного выполнения.

**Назначение:**

Диаграммы показывают алгоритм обработки заявки на вывод средств: проверку баланса, проверку лимитов, успешный вывод и отклонения.

**Действия и условия:**

| № | Действие / условие | Ответственный | Описание в тексте |
|---|---|---|---|
| 1 | Получить заявку на вывод | Система | Старт обработки заявки |
| 2 | Баланс >= сумма? | Система | Проверка достаточности средств |
| 3 | Проверить лимиты | Система | Проверка лимитов операции |
| 4 | Лимит не превышен? | Система | Ветвление по лимиту |
| 5 | Выполнить вывод | Система | Успешное выполнение |
| 6 | Отправить уведомление | Система | Уведомление клиента |

**PlantUML: Activity Diagram с условиями**

```plantuml
@startuml
title Activity Diagram — Вывод средств
caption Проверка баланса и лимитов перед выполнением вывода

start
:1. Получить заявку на вывод;
if (Баланс >= сумма?) then (да)
  :2. Проверить лимиты;
  if (Лимит не превышен?) then (да)
    :3. Выполнить вывод;
    :4. Отправить уведомление;
  else (нет)
    :3a. Отклонить: лимит превышен;
  endif
else (нет)
  :2a. Отклонить: недостаточно средств;
endif
stop

legend right
  | Обозначение | Значение |
  | start / stop | Начало и конец процесса |
  | :action; | Действие |
  | if / else / endif | Условие и ветвление |
  | (да) / (нет) | Подписанные исходы условия |
endlegend
@enduml
```

**PlantUML: Fork/Join — все потоки должны завершиться**

```plantuml
@startuml
title Activity Diagram — Параллельные проверки заявки
caption Fork/Join: создание ордера возможно только после завершения всех проверок

start
:1. Получить заявку;
fork
  :2. Проверка KYC;
fork again
  :3. Проверка лимитов;
fork again
  :4. Резервирование средств;
end fork
:5. Создать ордер;
stop

legend right
  | Обозначение | Значение |
  | fork | Разделение на параллельные потоки |
  | fork again | Дополнительная параллельная ветка |
  | end fork | Join: ожидание всех потоков |
endlegend
@enduml
```

**PlantUML: Fork/Merge — достаточно одного потока**

```plantuml
@startuml
title Activity Diagram — Отправка уведомления
caption Fork/Merge: уведомление считается доставленным при успешной доставке в один канал

start
:1. Отправить уведомление;
fork
  :2. Email;
fork again
  :3. Push;
fork again
  :4. SMS;
end merge
:5. Уведомление доставлено;
stop

legend right
  | Обозначение | Значение |
  | fork | Разделение на параллельные потоки |
  | fork again | Дополнительная параллельная ветка |
  | end merge | Merge: достаточно одного завершившегося потока |
endlegend
@enduml
```

**Легенда:**

| Обозначение | Значение |
|---|---|
| `start` | Начало процесса |
| `stop` | Конец процесса |
| `:Действие;` | Шаг процесса |
| `if / else / endif` | Условие и ветвление |
| `fork / end fork` | Параллельные потоки с ожиданием всех веток |
| `end merge` | Объединение без ожидания всех потоков |
