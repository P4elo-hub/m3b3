# Data Standard Kit

Этот каталог раскладывает раздел `5.1. Модель данных` из `standarts_features (3).md` на рабочие слои для агентного pipeline.

Kit помогает проектировать и описывать модель данных при изменении структуры хранилища: реляционные СУБД (PostgreSQL, Oracle, MySQL и др.), а также документные и другие базы — с адаптацией терминов (`таблица` / `коллекция`, `FK` / `reference`), но с сохранением логики сущностей, атрибутов и связей.

## Зачем нужен kit

Модель данных часто описывается разрозненно: ER-диаграмма отдельно, DDL отдельно, связи — в комментариях. Kit задаёт единый артефакт для аналитика и разработчика.

| Слой | Зачем нужен | Что лежит здесь |
|---|---|---|
| `skills/` | Учит агента анализировать и проектировать модель данных. | `data-model-analysis/SKILL.md` |
| `spec-kit/` | Задает контракт валидного результата. | `data-model.schema.yaml` |
| `templates/` | Дает форму документа, которую агент заполняет. | `data-model.template.md` |
| `examples/` | Канонический пример: стандарт + секции kit (контекст, область, gaps). | `payment-processing-data-model-example.md` |
| `quality-gates/` | Проверяет, можно ли принимать результат. | `data-model-review.yaml` |

## Как использовать в runbook

Пример шага:

```text
Команда: описать модель данных.

Используй:
- skill: data-standard-kit/skills/data-model-analysis/SKILL.md
- spec: data-standard-kit/spec-kit/data-model.schema.yaml
- template: data-standard-kit/templates/data-model.template.md
- example: data-standard-kit/examples/payment-processing-data-model-example.md
- quality gate: data-standard-kit/quality-gates/data-model-review.yaml

Выход:
- 03-Data/<feature>-data-model.md
```

## Когда применять

Применяй этот kit, если фича:

- добавляет, изменяет или удаляет сущности хранилища;
- меняет атрибуты, типы, nullable, ограничения или индексы;
- вводит или меняет связи между сущностями;
- требует сводную (junction) таблицу для связи N:M;
- затрагивает существующую ER-модель.

Не применяй этот kit для:

- DDL/DML-миграций и rollback — это отдельный будущий kit по `5.2. Миграции данных`;
- маппинга полей интеграций — это `5.3. Маппинг данных` / `integration-standard-kit`;
- чисто прикладной логики без изменения хранилища.

## Граница с другими kits

| Kit | Когда использовать вместо или вместе |
|---|---|
| `requirements-standard-kit` | Сначала собрать требования; data kit уточняет сущности, атрибуты и связи |
| `use-case-standard-kit` | Use Case описывает поведение; data kit описывает, что и как хранится |
| `integration-standard-kit` | Контракт обмена; data kit — персистентная модель |
| `uml-diagram-standard-kit` | ER-диаграмма в PlantUML — часть data kit; sequence/activity — для поведения |

## Главная идея

- Skill говорит агенту, как думать, какие уточнения задать и как не выдумывать типы и ограничения.
- Spec говорит, каким должен быть результат.
- Template говорит, куда писать.
- Example показывает норму качества.
- Quality gate решает, можно ли считать модель данных готовой.

## Clarification-First

По умолчанию skill **не рисует финальный артеfact сразу**. Сначала задаёт уточняющие вопросы с обязательными вариантами ответа:

- **A)** Пришлю контекст — DDL, ER, ORM, contract, requirements
- **B)** Спроектируй сам с нуля — `DESIGN-DATA-*`
- **C)** Частично знаю — confirmed + `GAP-DATA-*` / `ASM-DATA-*`
- **D)** Пока не знаю — только `GAP-DATA-*`

Финальный артеfact допустим без уточнений только если пользователь явно просит draft, прислал evidence или выбрал greenfield design.

## Канонический пример vs шаблон

- `examples/payment-processing-data-model-example.md` — канонический пример на основе `standarts_features (3).md`: блоки ER-диаграммы, сущностей, связей и миграций совпадают со стандартом; дополнительно включены секции kit: `Контекст изменения`, `Сводные таблицы (junction)`, `Область изменения`, `Gaps и допущения`.
- `templates/data-model.template.md` — форма **финального артефакта агента** для kit по `5.1`: без `СТД-*` в заголовках секций, с обязательными секциями `Область изменения` и `Gaps и допущения`, без блоков миграций (они — в будущем kit `5.2`).
