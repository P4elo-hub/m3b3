# DocIntel

AI-ассистент для извлечения, генерации и поиска по проектной документации на основе Markdown-файлов и AI-моделей.

## Назначение

Централизованное взаимодействие с технической документацией системного аналитика.

## Импорт из Confluence

Выгрузка пространств через REST API, конвертация страниц в единый Markdown с сохранением заголовков, таблиц и вложений.

Обратная синхронизация Markdown → Confluence доступна для пространств с меткой `docintel-sync`.

## Формат и хранение

Единый формат документов: Markdown с YAML front matter (`title`, `author`, `version`, `tags`, `source`).

Документация хранится в git-репозитории с версионированием и историей изменений.

## Поиск

Инструмент `search_kb` ищет релевантные фрагменты по ключевым словам в файлах `app/data/docs/*.md`, разбитых по заголовкам.

## Создание и редактирование

Создание документов: через интерфейс DocIntel или из шаблонов — BRD, FRD, API-spec, ADR.

Редактирование: изменения сохраняются в Markdown-файлах, доступен предпросмотр и diff перед коммитом.

## Шаблоны

### BRD

Разделы: «Цели», «Заинтересованные стороны», «Бизнес-требования», «Ограничения», «Критерии приёмки».

### FRD

Разделы: «Функциональные требования», «Нефункциональные требования», «Диаграммы», «Зависимости».

### API-spec

OpenAPI 3.0: для каждого endpoint — метод, путь, параметры, коды ответов, примеры.

## Глоссарий

- **BRD** — Business Requirements Document
- **FRD** — Functional Requirements Document
- **ADR** — Architecture Decision Record
- **SA** — System Analyst

## Регламент

Каждое функциональное требование имеет уникальный ID (`FR-NNN`), версию и автора в front matter.
