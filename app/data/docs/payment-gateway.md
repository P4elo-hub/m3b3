# Платёжный шлюз

Документация проекта платёжного шлюза.

## Функциональные требования

### FR-001 Авторизация платежа

Авторизация платежа должна завершаться не дольше 3 секунд при нагрузке до 500 TPS.

### FR-002 Способы оплаты

Поддержка интеграции с СБП (Система быстрых платежей) и картами МИР.

## Нефункциональные требования

### NFR-001 Доступность

Доступность сервиса 99.9%, RPO 15 минут, RTO 1 час.

## API

### POST /v1/payments

Создание платежа. Обязательные поля:

- `amount` (decimal)
- `currency` (ISO 4217)
- `merchant_id` (uuid)

### GET /v1/payments/{id}

Статус платежа. Возможные значения `status`: `pending`, `authorized`, `failed`, `refunded`.

## Архитектурные решения

### ADR-007 Event-driven обработка

Выбран event-driven подход с Kafka для асинхронной обработки уведомлений от банков-эквайеров.
