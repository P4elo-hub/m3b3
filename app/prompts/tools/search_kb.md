# search_kb — семантический поиск по проектной документации {{ service_name }}

Используй этот инструмент, когда пользователь спрашивает о **содержимом документации**: требованиях, API, процессах, форматах, интеграциях, шаблонах, глоссарии.

## Когда вызывать

- Вопросы про требования (BRD, FRD, user story), спецификации API, архитектурные решения (ADR).
- Запросы об импорте из Confluence, API, требованиях, шаблонах документов в `app/data/docs/`.
- Поиск по проекту: «какие поля у POST /payments», «требования к авторизации», «как экспортировать из Confluence».
- Любой запрос, где ответ должен опираться на актуальную проектную документацию, а не на общие знания модели.

## Когда НЕ вызывать

- Приветствия, благодарности, уточнения без новых фактов.
- Вопросы вне темы документации и системного анализа — ответь по правилам system prompt без поиска.
- Запросы на написание нового документа с нуля — сначала уточни тип документа; поиск нужен, чтобы не дублировать существующие материалы.

## Как формировать query

- Короткая фраза с ключевыми словами: «импорт Confluence Markdown», «POST payments API», «шаблон BRD», «глоссарий FRD».
- На русском или английском — база содержит термины на обоих языках.
- Один запрос за раз; при необходимости вызови повторно с другими ключевыми словами (синонимы, ID требования, endpoint).

## Формат результата

Инструмент возвращает до 5 фрагментов из Markdown-файлов (`app/data/docs/`) с указанием файла и заголовка раздела, или сообщение «ничего не найдено». Цитируй найденное, не дополняй из памяти.
