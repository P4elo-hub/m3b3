# write_feature_doc — написание документации новой фичи

Вызывай, когда пользователь просит **описать / задокументировать новую фичу** (даже короткий бриф).

## Параметры

| Поле | Описание |
|------|----------|
| `feature_brief` | Краткое описание фичи (обязательно) |
| `section_id` | `full` — полный документ по shablon (по умолчанию) |
| `protocol` | `Async` / `REST` / `gRPC` для интеграций |
| `feature_name` | Название фичи для заголовка `# ...` |

## После вызова — формат ответа пользователю

Верни **один** Markdown-файл со структурой shablon:

```text
# [Название фичи]
## 1. Бизнес-требования
### 1.1. Цель
### 1.2. Процесс/Сервис AS IS
### 1.3. Процесс/Сервис TO BE
## 2. Ограничения и допущения
## 4. Функциональные требования
#### 4.1.2. Асинхронное событие ...
## 5. Нефункциональные требования
### 5.1. Требования безопасности
### 5.2. Надёжность и производительность
#### 5.3.1. Требования к логированию
#### 5.3.2. Требования к мониторингу
### 5.4. Требования к настройкам
```

**Запрещено:** «Пакет методологии», «Раздел X | kit», служебные строки kits (`> **Kit`, `> **Skill`, `> **Покрывает стандарты`), code fence вокруг всего ответа.

**Язык:** финальный документ — **на русском**. Skills/templates на английском — только справка, не копируй.

Заполни скелет из tool; AS IS = сейчас, TO BE = после фичи.

**Use Case (обязательно в 1.2 и 1.3):** таблица Use Case, предусловия, постусловия. Основной сценарий — **не простой список**, а шаги с ветвлениями:

```
ЕСЛИ [BR-01] условие
  ТО сценарий продолжается с шага 3
ИНАЧЕ
  ПЕРЕЙТИ К альтернативному сценарию 2a
```

Альтернативы — с `Возврат к шагу N` или `Сценарий завершается неуспешно`. Минимум один альтернативный сценарий. Диаграммы plantuml — опционально.

**NFR (раздел 5):** все подразделы из скелета заполняй полностью по kit template (`nfr-standard-kit`, `observability-standard-kit`) — таблицы безопасности, производительности, надёжности, логирования, метрик, конфигурации. Логирование и мониторинг — **только** под `## 5. Нефункциональные требования`, не заменяй таблицы списком из 2–3 пунктов.
